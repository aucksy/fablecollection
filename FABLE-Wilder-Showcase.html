Five nature watch faces built on what nature and Watch Face Format genuinely share: things that change because time changes. The sun travels because the hour advances. The scene reacts to the clock and calendar — sun position, day/night, seasons — because those are the data sources WFF actually has.

How to read the feasibility badges
NATIVE

Built into Watch Face Format itself and drawn by a mechanism named in the row — time-driven transforms, time-range gates, native hand rotation, static tinted assets, arc fills. Ships in the face with no dependencies.

PROVIDER

Data arrives through standard Wear OS complication providers. Defaults are restricted to the WFF v1 system-provider list (SUNRISE_SUNSET is one!); heart-rate and weather can be user-added but can never be a default.

SIMULATED

Shown live in this prototype with generated values so you can judge the design. The real face binds the same slot to a real provider. Nothing renders here that the real WFF face couldn't draw.

CUSTOM

Would need our own complication provider or companion app. In WILDER this badge appears only on optional, non-signature extras — no face depends on one to ship.

⚠ build cost — rows carrying this note are possible-but-costly: the mechanism is real, but it multiplies elements/assets or needs one dedicated verification round. Costed here so nothing is discovered after building.

Designed inside the format — the natural walls WILDER was built against
Drifting clouds, waves, rain, fireflies

WFF has no particle systems and no ambient motion — only clock-rate rotation and data-driven fills. Scenes are still. The sun moves because time moves; that is the whole animation budget, spent well.

Weather-reactive scenery

WFF v1 has NO weather source, and even v2's can be absent. Weather is user-added complication data in a slot — the landscape never pretends to know if it's raining. Staying v1 keeps Wear OS 4 devices.

Moon-phase arithmetic

Phase maths needs the shelved % operator and epoch-day sources that don't exist. Moon phase is a provider slot only; the drawn moon is a time-of-night marker, not a phase display.

Organic vector shapes

PartDraw draws lines, rectangles, ellipses and arcs — no freeform paths. Mountains, branches, petals and contours ship as static tinted PNG assets, each costed against the 10 MB ambient gate (all are small, mostly-transparent, single-tint).

True 24-hour sky rotation

Native smooth rotation exists at hand rates (12 h for an hour hand). PLANISPHERE uses a stylised 12-hour sky natively; a true 24-h rotation is a costed option via the proven minute-stepped angle Transform.

Wind, temperature or heart-rate driving motion

Data drives FILLS (the proven arc transforms), never movement. Growth, tide and bloom metaphors are fills and gates — not kinetics.

Hemisphere / location awareness

No location source exists in the format. ALMANAC's automatic seasons use northern-hemisphere month mapping, with a manual season picker (the proven ListConfiguration switch) for everyone else.

True sunrise/sunset geometry

No astronomy source in v1. Drawn sun paths use fixed equinox presets (the shipped Solstice compromise); real times arrive as SUNRISE_SUNSET provider text — a genuine v1 default.

About this prototype
This page is a design simulator, not a build. The clock is your real local time and all complication values are generated placeholders. No Wear OS project, APK or production code exists for WILDER yet — that is a deliberate Phase-1 gate. Like the LEDGER and ARMATURE showcases, this simulator is feasibility-locked: the sun and sky move only at clock rates, day/night and seasons switch by instant substitution through proven gates, nothing drifts or sways, and every rendered element corresponds to a named WFF v1 mechanism already proven on a real watch in Series 01. If it renders here, it can ship.
