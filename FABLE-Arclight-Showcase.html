How to read the feasibility badges
NATIVE

Built into Watch Face Format itself — time, date, battery, step count, moon phase arithmetic, timezone clocks, expressions, animations. Ships in the face with no dependencies.

PROVIDER

Data arrives through standard Wear OS complication providers (Health services, Weather, Calendar, Clock…). The user slots them in the system editor; availability can vary by watch and installed apps.

SIMULATED

Shown live in this prototype with generated values so you can judge the design. The real face binds the same slot to a real provider.

CUSTOM

Would need our own complication provider or companion app (e.g. live sunrise/sunset geometry). Flagged honestly — none of the five faces depends on one to ship.

About this prototype
This page is a design simulator, not a build. Everything you see is rendered live in the browser: the clock is your real local time, animations run at simulator rate, and all complication values are generated placeholders. No Wear OS project, APK or production code exists yet — that is a deliberate Phase-1 gate. Every capability claimed in the spec sheets is labelled NATIVE / PROVIDER / SIMULATED / CUSTOM so the build scope is honest before a single line of watch code is written.
