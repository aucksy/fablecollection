Afterglow

Five watch faces from the light of old displays — arcade pixels, phosphor terminals, neon tubes, bedside LEDs and the pocket-calculator LCD. Watch Face Format cannot glow, so nothing here glows: neon is a lighter line inside a darker line, phosphor is a palette, a ghost 88:88 is free static text, and digits substitute instantly — which is exactly how real segment displays behave. The format's bluntest limits are this series' native behaviour.

5 faces · one APK each 25 colour themes · ≤5 roles each WFF v1 · Wear OS 4+ · widest reach per-face complications · ≤8 slots pure-black toggle · every face live simulator · your local time

Ghost segments are free

A static 88:88 in a seven-segment font behind the live time is the cheapest richness in the whole collection — pure static text, zero expressions, pure LCD soul.

Instant is authentic

WFF digits substitute instantly, with no transition frames. That is precisely how LCD, LED and VFD displays have always behaved. For once the wall IS the aesthetic.

Neon without glow

No blur, bloom or blend modes exist in v1. A tube is a lighter core stroke laid inside a darker casing stroke — flat, crisp, and it reads as neon at arm's length.

OLED economics

Black grounds with thin bright strokes and sparse type: dramatic interactive faces that walk under the 15% always-on pixel gate without dieting.

How to read the feasibility badges
NATIVE

Built into Watch Face Format itself and drawn by a mechanism named in the row — static text and strokes, threshold-gated duplicate cells, time-range gates, native hand rotation, TimeText. Ships in the face with no dependencies.

PROVIDER

Data arrives through standard Wear OS complication providers. Defaults are restricted to the WFF v1 system-provider list; heart-rate and weather can be user-added but can never be a default.

SIMULATED

Shown live in this prototype with generated values so you can judge the design. The real face binds the same slot to a real provider. Nothing renders here that the real WFF face couldn't draw.

CUSTOM

Would need our own provider, companion app, or an unproven runtime mechanism. In AFTERGLOW this badge appears only on rejected or optional non-signature extras — no face depends on one to ship.

⚠ build cost — rows carrying this note are possible-but-costly: the mechanism is real, but it multiplies elements/assets or needs one dedicated verification round. Costed here so nothing is discovered after building.

Designed inside the format — the retro walls AFTERGLOW was built against
Glow, bloom, halos, light bleed

WFF v1/v2 has no filter pipeline — no blur, no shadow, no blend modes. Neon is layered flat strokes: a lighter core inside a darker casing. A soft halo is impossible and never appears in this simulator.

Phosphor trails, decay, afterimages

Nothing in the format can fade or persist — elements are visible or they are not. The "afterglow" is a palette and a ghost underlay, never a motion effect.

Flicker, scan roll, CRT wobble

No shader, no animated texture, no opacity animation. Scanlines are ONE static texture asset; the screen never rolls, hums or jitters.

Blinking colon / blinking cursor

A 1 Hz blink needs [SECOND] % 2 — and the % operator died silently on a real watch (shelved). Every colon and cursor in AFTERGLOW is steady, by design. A 2-frame animated-image blink exists in theory but is runtime-unproven — listed as a costed non-signature option only.

Marching sprites, demo-scene motion

The only motion in the format is clock-rate rotation and data-driven fills. Sprites stand still; score digits change because the data changed — like a real cabinet between plays (attract mode is the one thing we don't simulate).

Low battery turns the display red

No conditionals exist in any colour attribute, ever. State is shown by discrete threshold-gated CELLS (each its own element with ONE comparison) — bars drain like real segment meters instead of recolouring.

Letter-spaced terminal type

Fonts have no letterSpacing attribute in WFF. Segment, pixel and terminal fonts are naturally tabular and tight-set — the one type family where the wall costs nothing.

Press-to-backlight

TAP triggers can only play pre-rendered image sequences — they cannot recolour the scene, and an Indiglo-style flood is a theme, not an event. The backlit look ships as a colour theme (SEGMENT's "Indiglo") the user selects.

About this prototype
This page is a design simulator, not a build. The clock is your real local time and all complication values are generated placeholders. No Wear OS project, APK or production code exists for AFTERGLOW yet — that is a deliberate Phase-1 gate. Like the LEDGER, ARMATURE and WILDER showcases, this simulator is feasibility-locked: no glow, blur or blend appears anywhere; nothing blinks, trails or flickers; the seven-segment digits drawn here are exactly what a static DSEG font file renders on the watch; and every rendered element corresponds to a named WFF v1 mechanism already proven on a real watch in Series 01. If it renders here, it can ship.
