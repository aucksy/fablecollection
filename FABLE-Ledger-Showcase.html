Five typographic watch faces where the numerals are the face — no hands to hide behind, no dashboard cosplay. Named after the classical type sizes, from Canon down to Nonpareil. Type is the interface

Weight, scale, position and colour do all the work. The strongest levers WFF actually has — not the ones it doesn't.

No motion cosplay

WFF has no digit transitions, no property animation, no wake trigger for vectors. LEDGER's identity is composition, so nothing here pretends to move.

Ship-safe by construction

Every signature maps to an on-wrist-proven pattern: alpha gates, native Sweep, Transform arc fills, duplicate-and-gate colour states.

OLED economics

Ink-on-black typography is the cheapest thing a watch can draw. Sparse thin-weight AOD variants keep every face under the 15% pixel gate.

How to read the feasibility badges
NATIVE

Built into Watch Face Format itself and drawn by a mechanism named in the row — TimeText, PartText tokens, alpha-gate Transforms, arc fills, the seconds Sweep. Ships in the face with no dependencies.

PROVIDER

Data arrives through standard Wear OS complication providers. Defaults are restricted to the WFF v1 system-provider list; heart-rate and weather can be user-added but can never be a default.

SIMULATED

Shown live in this prototype with generated values so you can judge the design. The real face binds the same slot to a real provider. Nothing renders here that the real WFF face couldn't draw.

CUSTOM

Would need our own complication provider or companion app. In LEDGER this badge appears only on optional, non-signature extras — no face depends on one to ship.

⚠ build cost — rows carrying this note are possible-but-costly: the mechanism is real, but it multiplies elements or needs one dedicated on-wrist verification round. Costed here so nothing is discovered after building.

Designed inside the format — the walls LEDGER was built against
Letter-spaced, airy numerals

WFF fonts have no letterSpacing attribute. So every face sets type tight — the aesthetic is dense masthead lockups, not tracked-out captions. (Spacing separate PartText glyphs by hand is possible but costs one element per glyph; not used anywhere.)

Flip / slide / fade digit transitions

No vector property animation and no time-since-wake source exist. Digits change by instant substitution, exactly as WFF renders them. The only motion in the series: the native seconds Sweep and proven data-driven arc fills.

Arbitrary rendered words

String-literal Template parameters fail at runtime (numeric %d only). Every word on these dials is either a native token ([DAY_OF_WEEK_S], [MONTH_S], [AMPM_STRING]), provider text, or static <Text> — which is why BREVIER's grammar is a fixed grid of static words, lit by gates.

Colour logic in colour attributes

No ternaries in any colour attribute, and a ColorConfiguration selection can never be branched on. Every colour-by-state element ships as duplicated copies alpha-gated on a Boolean/List config — the pattern that fixed Paper Dawn in Series 01.

Per-digit arithmetic (h % 12 tricks)

The % operator died silently on-wrist in Series 01 and stays shelved. No face here splits digits by modulo; hour mapping uses paired equality gates or the [HOUR_1_12] token (verified against the XSD before build).

Glow, blur and blend modes on type

Blend modes are WFF v3+, and text has no filter pipeline. LEDGER's type is flat by design — crisp ink, no halos. Depth comes from draw order and colour roles.

One-tap layout presets (v1)

Flavors are WFF v2+. The presets in this simulator correspond on-watch to saved slot/toggle states in the system editor (every slot can be set Empty). Staying v1 keeps Wear OS 4 devices; Flavors can arrive later as a v2 refresh without redesign.

Giant black glyphs in always-on

Ambient must stay ≤15% lit pixels. Every AOD variant here is a structural thin-weight twin (base alpha 0 + Variant AMBIENT 255 — the exact Series 01 pattern) in fixed light ink, so even light themes stay readable on the black ambient canvas.

About this prototype
This page is a design simulator, not a build. The clock is your real local time and all complication values are generated placeholders. No Wear OS project, APK or production code exists for LEDGER yet — that is a deliberate Phase-1 gate. Unlike a normal mock-up, this simulator is feasibility-locked: dial text carries no letter-spacing, digits never animate, colours never blend, and every rendered element corresponds to a named WFF v1 mechanism already proven on a real watch in Series 01. If it renders here, it can ship.
